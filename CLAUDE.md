# Frontier Alpha

AI-powered cognitive factor intelligence platform — institutional-grade portfolio analysis with explainable AI, self-improving belief systems, and real-time market streaming.

## Architecture

Monorepo with two top-level packages:

```
frontier-alpha/
  src/          — Fastify server (Intelligence + Compute + Data layers)
  client/       — React 19 SPA (Interface layer)
```

### Server (`src/`)

| Subsystem | Key Files | Purpose |
|-----------|-----------|---------|
| **Factor Engine** | `factors/FactorEngine.ts` | 76 factor exposures across 12 groups (6 code categories) |
| **CVRF** | `cvrf/CVRFManager.ts`, `BeliefUpdater.ts`, `EpisodeManager.ts`, `PersistentCVRFManager.ts` | Episodic belief learning system |
| **Earnings Oracle** | `earnings/EarningsOracle.ts` | Calendar, forecasts, beat rates, expected moves |
| **Cognitive Explainer** | `explanation/ExplanationService.ts` | GPT-4o + template dual-mode explanations |
| **Portfolio Optimizer** | `optimizer/` | Monte Carlo — max Sharpe, min variance, risk parity |
| **Backtest** | `backtest/` | Walk-forward engine with CVRF integration |
| **Risk** | `risk/RiskAlertSystem.ts` | 11 alert types, real-time monitoring |
| **Market Data** | `data/MarketDataProvider.ts` | Polygon.io WebSocket + Alpha Vantage + caching |
| **Trading** | `trading/BrokerAdapter.ts` | Alpaca paper + live trading |
| **ML** | `ml/` | Regime detection, neural factor models |
| **Notifications** | `notifications/PushService.ts` | VAPID Web Push + SSE |
| **Auth** | `middleware/AuthMiddleware.ts` | JWT + API key + Supabase RLS |

See `ARCHITECTURE.md` for the full file-level map (generated by arch-scanner).

### Client (`client/src/`)

| Layer | Details |
|-------|---------|
| **Pages** | 19 views — Dashboard, Portfolio, Factors, CVRF, Earnings, Options, Tax, Trading, Backtest, ML, Social, Settings, and more |
| **Components** | ~95 React 19 components across 19 domains |
| **State** | 6 Zustand stores — `portfolioStore`, `quotesStore`, `alertsStore`, `authStore`, `themeStore`, `dataSourceStore` |
| **Real-time** | WebSocket → SSE → Polling progressive fallback |
| **Design system** | CSS variable–based (see `DESIGN-SYSTEM.md`) |

## Development

```bash
# Install dependencies
npm install && cd client && npm install && cd ..

# Start dev (API port 3000 + client port 5173)
npm run dev:all

# Individual services
npm run dev              # API server only
npm run client:dev       # Client only

# Testing
npm test                 # Server tests (watch mode)
npm run test:unit        # Server unit tests (single run)
npm run test:all         # All tests — server + client (205 total)
npm run test:coverage    # Coverage report

# Build & lint
npm run build            # Production build
npm run lint             # ESLint

# Database
npm run db:migrate       # Apply Supabase migrations (supabase db push)

# Optional Python ML engine
npm run ml:start         # uvicorn on port 8000
```
